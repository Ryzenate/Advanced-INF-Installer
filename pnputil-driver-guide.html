Windows Driver Infrastructure

PnPUtil
Driver Analysis
Pipeline

A systematic approach to enumerating INF files across driver categories, evaluating hardware ID compatibility, and determining system applicability without destructive operations.

// OVERVIEW

Pipeline Architecture

Five-stage pipeline from raw INF discovery to ranked, applicable driver list.

// END-TO-END FLOW
// STEP 01

Enumerate All INF Files

Recursively walk the driver root directory, collecting every .inf file regardless of subfolder depth. Export a master manifest CSV for reference throughout the pipeline.

// FILESYSTEM TRAVERSAL MODEL
PowerShell — Enumerate & Export
# Collect every INF found under a root driver directory $driverRoot = "C:\Drivers" $allInfs = Get-ChildItem -Path $driverRoot -Recurse -Filter "*.inf" | Select-Object FullName, DirectoryName, Name, LastWriteTime, Length # Export master manifest CSV $allInfs | Export-Csv -Path "C:\Temp\AllINFs.csv" -NoTypeInformation Write-Host "Found $($allInfs.Count) INF files"
// STEP 02 + 03

Parse INF Hardware IDs & Collect System IDs

Extract hardware IDs embedded in each INF's [Models] section, and simultaneously enumerate all hardware IDs reported by devices on the running system.

// INF INTERNAL STRUCTURE — KEY SECTIONS
INF Bus Prefixes Targeted
PCI\   USB\   ACPI\
HID\   HDAUDIO\
ROOT\   STORAGE\
SCSI\   SD\
System Query Method
DEVPKEY_Device_HardwareIds

Queries every PnP device present or historically connected. Returns all alternative hardware IDs per device for broadest matching coverage.
Non-Destructive
INF parsing reads files only — no writes to the Driver Store, no registry changes, no staging. Safe to run on production systems without any rollback concern.
PowerShell — Collect System Hardware IDs
# Get all hardware IDs present on THIS system $systemHWIDs = Get-PnpDevice | ForEach-Object { (Get-PnpDeviceProperty -InstanceId $_.InstanceId ` -KeyName 'DEVPKEY_Device_HardwareIds').Data } | Where-Object { $_ } | Sort-Object -Unique
// STEP 04

Match & Score Applicability

Intersect each INF's extracted hardware IDs against the system's live hardware ID set. Score by match count to produce a ranked applicability list.

// MATCHING LOGIC — PER INF
PowerShell — Full Match & Score Loop
$results = foreach ($inf in $allInfs) { $content = Get-Content $inf.FullName -ErrorAction SilentlyContinue # Extract all hardware ID-like strings from INF $infHWIDs = $content | Select-String ` -Pattern '\b(PCI\\|USB\\|ACPI\\|HID\\|HDAUDIO\\|ROOT\\|STORAGE\\|SCSI\\)[^\s,]+' ` -AllMatches | ForEach-Object { $_.Matches.Value } | Sort-Object -Unique # Find overlap with system hardware IDs $matches = $infHWIDs | Where-Object { $systemHWIDs -contains $_ } [PSCustomObject]@{ INF = $inf.FullName INF_HWIDCount = $infHWIDs.Count MatchCount = $matches.Count Applicable = ($matches.Count -gt 0) MatchedHWIDs = ($matches -join "; ") } } $results | Export-Csv -Path "C:\Temp\INF_Applicability.csv" -NoTypeInformation
// STEP 05

PnPUtil Staging Confirmation

Optionally stage matched INFs into the Windows Driver Store via PnPUtil as a secondary confirmation layer. Windows will reject INFs whose hardware IDs have no match on the system.

Staging adds drivers to the DriverStore permanently until removed. Use /delete-driver cleanup afterward if you don't want non-applicable drivers to persist.
// PNPUTIL STAGING DECISION TREE
PowerShell — Stage & Capture Results
foreach ($inf in $allInfs) { $result = pnputil /add-driver "$($inf.FullName)" 2>&1 [PSCustomObject]@{ INF = $inf.FullName Result = $result -join " " Applicable = $result -match "Driver package added successfully" } } | Export-Csv -Path "C:\Temp\StagedDriverResults.csv" -NoTypeInformation
// REFERENCE

Decision Matrix

Use this table to classify each INF after running the pipeline. Prioritize double-confirmed matches; skip superseded drivers.

Condition MatchCount PnPUtil Stage Verdict Action
HWID found on system, staged OK > 0 Success ✅ STRONG MATCH Install / keep staged
HWID found on system, not staged > 0 N/A ✅ APPLICABLE Safe to install
No HWID match, no Model IDs in INF 0 Varies ⚠ MANUAL REVIEW May be filter/service driver

// COMPONENTS

Key Components & Tools

Each component in the pipeline and its specific role in the analysis workflow.

Get-ChildItem -Recurse
PowerShell cmdlet that walks the entire directory tree from a root path, filtering for .inf files at any depth. Produces the master file manifest.
Get-PnpDevice + Property
Queries the Windows PnP manager for all devices and retrieves their full hardware ID arrays via DEVPKEY_Device_HardwareIds. Builds the system ground truth.
Select-String (Regex)
Scans INF file content with a pattern matching known bus-prefix formats. Extracts all hardware IDs without requiring an INF parser library — works on any PowerShell version.
pnputil /add-driver
Microsoft's built-in driver store tool. Stages an INF into the DriverStore; Windows rejects incompatible packages automatically. Provides OS-level validation as a second confirmation pass.
pnputil /delete-driver
Removes a previously staged driver from the DriverStore. Should be run as a cleanup step for any INFs staged during analysis that were determined non-applicable.
Export-Csv / Import-Csv
Persists pipeline state between stages. Allows the enumeration, parsing, and ranking phases to run independently and be reviewed or filtered by external tools or team members.
// STEP 06

Cleanup — Remove Non-Applicable Staged Drivers

After staging and analysis, remove any INFs that were staged but not applicable to avoid cluttering the DriverStore.

PowerShell — Cleanup Non-Applicable
# Load results and remove non-applicable staged drivers $nonApplicable = Import-Csv "C:\Temp\INF_Applicability.csv" | Where-Object { $_.Applicable -eq "False" } foreach ($entry in $nonApplicable) { pnputil /delete-driver $entry.INF /uninstall 2>$null } # Final ranked output of applicable drivers Import-Csv "C:\Temp\INF_Applicability.csv" | Where-Object { $_.Applicable -eq "True" } | Sort-Object { [int]$_.MatchCount } -Descending | Format-Table INF, MatchCount, MatchedHWIDs -AutoSize
